# Prompts utilizados

## Prompt 1 - Generación inicial

Actúa como Product Manager y Business Analyst.

A partir del PRD adjunto, realiza lo siguiente:

1. Genera entre 4 y 6 User Stories enfocadas en un MVP.
2. Incluye criterios de aceptación claros
3. Prioriza usando MoSCoW
4. Descompón la User Story más importante en tareas técnicas
5. Estima usando Story Points

### Resultado
- Se generaron User Stories completas con criterios de aceptación
- Incluyó backlog priorizado
- Se propuso un desglose técnico y estimaciones

### Problemas
- El resultado fue demasiado extenso
- Incluía mucho detalle técnico innecesario para un MVP
- Algunas partes tenían un enfoque más “enterprise” que práctico

---

## Prompt 2 - Generación de documento Markdown

Organiza toda esta información en un documento Markdown listo para entrega.

Debe incluir:
- Contexto breve
- User Stories
- Backlog priorizado (tabla)
- Desglose técnico de una User Story
- Estimación
- Conclusiones

Usa formato claro, conciso y orientado a un MVP real. Evita texto innecesario.

### Resultado
- Generó un documento estructurado y ordenado
- Facilitó la organización de la información en formato entregable
- Incluyó todas las secciones necesarias

### Problemas
- Mantuvo demasiado nivel de detalle técnico
- Fue necesario simplificar manualmente el contenido
- Algunas descripciones seguían siendo más complejas de lo necesario

---

## Conclusiones

El segundo prompt fue más efectivo porque permitió estructurar directamente la información en un formato listo para entrega, reduciendo el trabajo manual de organización.

Sin embargo, fue necesario realizar ajustes para simplificar el contenido y adaptarlo a un MVP real.

El uso de prompts con:
- rol definido
- instrucciones claras
- estructura esperada

mejoró significativamente la calidad de las respuestas.

Además, la iteración entre prompts permitió pasar de un resultado más genérico a uno más estructurado y útil para un contexto real de desarrollo.
